# {{PROJECT NAME}}

Copy this `_TEMPLATE` folder, rename it to the project, and fill it. This is what NUNU reads before doing
any work on this project — so the richer it is, the more Frank-grade and specific the output.

## Drop these in
- **`guidelines.md`** — the project's marketing guidelines: positioning, key messages, do/don't,
  mandatories, any legal/REA/compliance lines that MUST appear.
- **`brand-voice.md`** — tone + voice for this project (how it should *sound*).
- **`design/`** — the **Claude Design project archive** (Yana exports it; drop the files here). Colours,
  typography, imagery, logo lockups, hero shots — NUNU pulls the visual language from here.
- **`briefs/`** — one file per campaign: goal (what buyer action), audience segment, the project moment,
  and any dated/price details (NUNU never invents these).
- **`facts.md`** (optional but gold) — the hard, checkable facts: address/locale, amenities, stages,
  prices, timelines, key selling points. NUNU draws specifics from here instead of guessing.

## NUNU saves here
- `campaigns/` — finished EDMs (`.html`) + social/activation idea sets, named + dated.

The rule: **no material = generic output = failure.** If something's missing, NUNU flags it to Yana
rather than inventing.
